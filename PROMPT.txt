在已有 resolve 字段批量加载能力的基础上新增 DataLoader 复用与缓存层：同一个 Loader 在一次 Resolver 全树解析过程中，针对同一组入参只发起一次底层 IO 调用，结果按入参列表原序映射回调用方；新增 LoaderCache 抽象提供按 key 去重的内存缓存，TTL 默认 60 秒可配置，跨 Resolver 实例之间不共享缓存实例；Loader 内部需要区分"冷启空载"与"热命中"两种状态并提供指标回调以便上层接入 Prometheus。在已有 ER Diagram 基础上新增跨实体字段映射声明：Entity 之间除了 Relationship 关系连线之外，允许声明纯字段映射（两个 Entity 各自有同名字段时自动复用该字段加载的 Loader 产物，不再单独发起请求），并且在 ER Diagram 渲染时把字段映射作为虚线箭头呈现，hover 时显示源与目标的字段类型签名。在已有 Collector 向上聚合机制的基础上新增 Collector 合并策略：同一 Collector 名出现在多个并列分支下时默认按遇到顺序拼接结果，提供 union、intersect、first、last 四种合并模式由调用方在 Collector 实例上声明；Collector 在父节点取值时不再区分来自哪个子节点，合并在底层完成而不是在 post 方法里手工去重。在已有 GraphQL SchemaBuilder 基础上新增 Schema 增量更新能力：调用方通过 SchemaBuilder.patch(schema) 提交局部 Schema 修改（新增 type 字段、新增 query 节点、新增 input 类型），框架把 patch 与已有 Schema 求并集后输出新 Schema；Schema 冲突时按"新增优先于覆盖、覆盖优先于删除"规则给出警告但保留 patch 一侧的版本。在已有 MCP 多应用管理能力的基础上新增 AppConfig 热加载：MCP 启动时从一个 YAML 配置文件读 AppConfig 列表（包含 name、entity_set、loader 路径），文件修改并发送 SIGHUP 后 MCP 自动重新装载这份配置并把新增的 App 注册到 MultiAppManager，已经注册的 App 在重载期间继续服务现有请求而不中断。
